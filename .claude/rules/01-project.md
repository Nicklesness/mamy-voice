# Правила проекта mamy-voice

## Стек и конвенции

- Next.js App Router (НЕ Pages Router). Все роуты в `src/app/`.
- TypeScript strict mode. Никаких `any` без явной причины.
- Tailwind CSS 4 для стилей. Никаких CSS-модулей или styled-components.
- Компоненты — функциональные, с хуками. Никаких классовых компонентов.
- `"use client"` — только там, где реально нужен клиентский JS (запись аудио, плеер, интерактив).
- Server Components по умолчанию для страниц без интерактива (онбординг, каталог книг).

## Файловая структура

- Компоненты UI: `src/components/ui/` (кнопки, карточки — переиспользуемые)
- Фичевые компоненты: `src/components/` (VoiceRecorder, AudioPlayer — привязаны к бизнес-логике)
- API клиенты: `src/lib/` (elevenlabs.ts, books.ts)
- Хуки: `src/hooks/`
- Типы: `src/types/`

## API Routes

- Все API роуты в `src/app/api/`
- Используем Route Handlers (export async function POST/GET)
- ElevenLabs API ключ только на сервере (НИКОГДА не отдавать на клиент)
- Валидация входных данных в каждом route handler

## Mobile-first

- Все страницы должны быть удобны на мобильных (основной use case)
- Крупные touch-таргеты (min 44x44px)
- Плеер — особенно крупные кнопки (ребёнок будет нажимать)
- Тестировать на viewport 375px

## Качество аудио

- Формат записи: WebM/Opus или WAV (что поддерживает браузер)
- Минимум 2 минуты записи для качественного клона
- Показывать визуальную обратную связь при записи (уровень громкости)
- Валидировать длительность перед отправкой

## UX принципы

- Минимум шагов до результата
- Понятные инструкции на каждом этапе (мама может быть не технически подкована)
- Прогресс-бар и примерное время при генерации
- Плеер — отдельный изолированный экран (безопасно дать ребёнку)
